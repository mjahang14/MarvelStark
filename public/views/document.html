Rest Api Documentation

1. Resource endpoint:

/api/getNFZ Type: GET Use this API to fetch geoJSON feature collection of No Flying Zone(NFZ).

2. Resource endpoint: /api/getRoute/

Type: POST Use this API to get Air Route for Drone. This API needs Source and Destination to be posted in GeoJSON(RFC 7946) point collection format. In response, route response will be sent in geoJSON using Point collection format. Returned path will be guaranteed shortest path

3. Resource endpoint:/api/populateNFZ/

Type: POST USe this API to draw No flying zone, NFZ data shall be posted in geoJSON format polygon. Poygon coordinates point should make a closed loop.

4. Resource endpoint: /api/getGrid/

Type: POST This api is used to get route grid data, this is developed to test routing algorithm for DEV team. User can use this to check if shortest route(in distance) is found avoiding all NFZ along path. (B) How it works
